# 章节回顾流程

## 回顾维度

### 1. 角色一致性
- 检查每个出场角色的行为是否与角色卡一致
- 对话风格是否前后统一
- 角色的情感反应是否符合其 emotional_tendency
- 角色成长/变化是否有逻辑基础

### 2. 情节连贯性
- 场景之间的过渡是否自然
- 时间线是否有矛盾
- 情节推进是否合理（无突兀转折）
- 前文埋下的线索是否在适当的时候被提及

### 3. 伏笔管理
- 列出本章所有涉及的伏笔及其状态
- 确认新埋的伏笔有明确的 pay_off_idea
- 确认已回收的伏笔标记为 paid
- 标记需要推进但尚未推进的伏笔

### 4. 秘密与信息不对等
- 检查每个秘密的知晓范围是否正确
- 确认没有角色"突然知道"之前不知道的秘密
- 检查 public_version 是否在角色间正确传播
- 标记可能的信息泄露风险

### 5. 节奏控制
- 章节长度是否合理
- 场景数量是否合适
- 高潮/转折的分布是否均衡
- 是否有拖沓或过于仓促的部分

## 输出格式
- 每个维度给出：状态（通过/有问题/需关注）
- 具体问题描述和位置
- 修改建议
