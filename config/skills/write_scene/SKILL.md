# 场景写作流程

## 前置检查
1. 确认当前场景的大纲（场景目标、参与者、时间地点）
2. 检查参与角色的角色卡——确保了解每个角色的声音指纹和知识范围
3. 确认当前伏笔状态——哪些需要推进，哪些需要回收

## 写作步骤
1. **开场**：根据场景时间设定开场氛围
2. **推进**：按场景目标推进情节，每个角色只展现其"知道"的内容
3. **对话**：严格遵循每个角色的说话风格，参考示例对话
4. **冲突/转折**：如果有预设的冲突点，按节奏展开
5. **收尾**：场景结束时有明确的推进或转折

## 知识边界规则
- 角色只能知道自己的知识范围（knowledge_scope）内的事情
- 角色只能知道"公开版本"的秘密，不能知道"真相"
- 如果角色从未与其他角色交流过某信息，不能表现出知道
- 叙述者可以暗示伏笔，但不能直接揭露

## 写作检查清单
- 每个角色的对话是否符合其说话风格？
- 角色行为是否符合性格特征（core_traits）？
- 是否有角色表现出了不该知道的信息？
- 场景是否推进了至少一个伏笔？
- 场景结尾是否有钩子吸引读者继续？
- 情感倾向是否正确体现在角色的决策中？
