# 伏笔布置流程

## 伏笔设计原则
1. **可见性分级**：
   - subtle（隐晦）：只有重读时才能发现的细节
   - visible（可见）：细心读者能注意到的线索
   - obvious（明显）：大多数读者能意识到的铺垫

2. **回收计划**：每个伏笔必须有明确的 pay_off_idea（回收方案）

3. **分散布置**：不要在同一场景中密集布置过多伏笔

## 操作步骤
1. **确定伏笔类型**：
   - 角色伏笔：某个角色的异常行为或隐藏动机
   - 物品伏笔：某个看似普通但后续关键的物品
   - 信息伏笔：某个看似随意但实际重要的信息
   - 关系伏笔：角色之间的微妙互动暗示未来关系变化

2. **选择 hint_level**：
   - subtle：适合长篇连载，让读者有"重读发现"的乐趣
   - visible：适合中期章节，保持读者好奇心
   - obvious：适合即将回收的伏笔，制造期待感

3. **编写伏笔内容**：自然嵌入场景叙述中，不突兀

4. **设定回收方案**：描述该伏笔在何处、如何被回收

5. **添加标签**：用 tags 标记伏笔相关的角色、事件、主题

## 检查清单
- 伏笔内容是否自然地融入场景？
- pay_off_idea 是否具体可行？
- hint_level 是否符合预期？
- 是否与已有伏笔有重复或冲突？
- 回收时机是否在合理的章节范围内？
